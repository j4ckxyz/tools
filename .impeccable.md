## Design Context

### Users
Developers, designers, and power users who want focused browser-based utilities. They use the tools to complete a concrete task quickly, without accounts, installations, or sending private working data to a service.

### Brand Personality
Practical, careful, and quietly confident. The interface should feel trustworthy and direct, with clear language about what a browser-only tool can and cannot do.

### Aesthetic Direction
A clean utility-workbench aesthetic that fits the existing Browser Tools design system. Both light and dark modes are first-class. Strong information hierarchy, restrained color, precise status treatments, and responsive layouts should make dense output easy to scan without turning the page into a generic dashboard.

### Design Principles
- Lead with the task and make the primary action obvious.
- State limitations and privacy behavior plainly; never overclaim capability.
- Use color and iconography semantically, with text labels for accessibility.
- Keep advanced controls available through progressive disclosure.
- Make results easy to scan, copy, and use on desktop or mobile.
